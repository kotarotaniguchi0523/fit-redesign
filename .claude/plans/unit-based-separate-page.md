# Plan: 完全単元別ページ（将来実装案）

## 概要

問題を単元ごとに分類し、年度をまたいで「オートマトンの問題だけ」を学習できる専用ページを作成する。

## 背景・課題

現在の問題データには「この問題はどの単元に属するか」というタグがない。2014年以降の統合試験（例：オートマトン＋符号理論）では、両方の問題が混在している。

単元学習に集中したいユーザーにとって、関係ない問題が混ざるのは非効率。

## 実装アイデア

### 1. 問題データへのタグ付け

```typescript
interface Question {
  id: string;
  number: number;
  text: string;
  // ... 既存フィールド
  unitTags?: string[];  // 新規追加: ["automaton"], ["ecc"], ["automaton", "ecc"] など
}
```

### 2. 新規ページ構成

```
/unit-study                    # 単元別学習ページ
  └── 単元選択（9単元）
       └── その単元の問題だけを全年度から抽出して表示
```

### 3. UI案

```
┌─────────────────────────────────────────────┐
│ 完全単元別学習モード                         │
├─────────────────────────────────────────────┤
│ 単元: [オートマトン ▼]                       │
├─────────────────────────────────────────────┤
│ オートマトンの問題（全12問）                 │
│                                             │
│ ■ 2013年 問題1                              │
│ ■ 2013年 問題2                              │
│ ■ 2014年 問題3 (統合試験より抽出)           │
│ ■ 2016年 問題1 (統合試験より抽出)           │
│ ...                                         │
└─────────────────────────────────────────────┘
```

## 必要な作業（実装時）

1. 全問題データにunitTagsを付与
2. 単元別ページコンポーネント作成
3. ルーティング追加
4. 問題フィルタリングロジック実装

## 優先度

低（現在の実装で基本的な学習は可能なため）

## メモ

- この機能は「試験対策」ではなく「単元の深い理解」を目的とするユーザー向け
- 問題のタグ付けは手作業が必要で工数がかかる
- 将来的にはAIで自動タグ付けも検討可能
